## 분석 결과

### 유형 판단
- **유형**: 신규 (아키텍처 재구성)
- **근거**: ScrollRect 래핑 → ScrollRect 코드 포크 방식으로 전면 재구성. 기존 기능은 유지하되 내부 구조가 근본적으로 변경됨

### 현재 아키텍처 분석

#### ScrollRect 의존 관계도
```
RecycleScroller
  ├─ [RequireComponent] ScrollRect  ← 외부 컴포넌트
  │   ├─ viewport (RectTransform)
  │   ├─ content (RectTransform)
  │   ├─ horizontal/vertical (축 제어)
  │   ├─ verticalNormalizedPosition / horizontalNormalizedPosition
  │   ├─ velocity (물리)
  │   ├─ inertia (관성)
  │   ├─ movementType (Elastic/Clamped/Unrestricted)
  │   ├─ onValueChanged → OnScrollRectScrolling() (프리팹 바인딩)
  │   └─ verticalScrollbar / horizontalScrollbar → 강제 null 처리
  │
  ├─ [SerializeField] RecycleScrollbar m_scrollbarRef (주축 1개)
  │   └─ Del = this (IRecycleScrollbarDelegate)
  │
  └─ IRecycleScrollbarDelegate 구현
      ├─ LoopScrollIsOn, IsLoopScrollable
      ├─ RealSize, ShowingSize, ViewportSize
      └─ ConvertRealToShow(), ConvertShowToReal()
```

#### `_ScrollRect` 참조 지점 (15곳)
| 파일 | 라인 | 용도 |
|------|------|------|
| `RecycleScroller.cs:35-36` | `Viewport`, `Content` 프로퍼티 | viewport/content 접근 |
| `RecycleScroller.cs:165-166` | `RealNormalizedScrollPosition` getter | 정규화 위치 읽기 |
| `RecycleScroller.cs:181,184` | `RealNormalizedScrollPosition` setter | 정규화 위치 쓰기 |

### 목표 아키텍처

#### 새로운 구조
```
RecycleScroller (MonoBehaviour + UIBehaviour)
  ├─ [내재화] ScrollRect 핵심 로직
  │   ├─ [SerializeField] m_viewport (RectTransform)
  │   ├─ [SerializeField] m_content (RectTransform)
  │   ├─ [SerializeField] m_movementType (Elastic/Clamped/Unrestricted)
  │   │   └─ OverwriteValue<MovementType> 적용:
  │   │       루프 ON → Unrestricted 강제, 루프 OFF → 사용자 설정값
  │   ├─ [SerializeField] m_elasticity, m_decelerationRate, m_scrollSensitivity
  │   ├─ [SerializeField] m_inertia
  │   ├─ 드래그 처리 (IBeginDragHandler, IDragHandler, IEndDragHandler)
  │   ├─ LateUpdate에서 관성/탄성/클램프 계산
  │   ├─ velocity 관리
  │   └─ onValueChanged 이벤트
  │
  ├─ 듀얼 스크롤바
  │   ├─ 주축 스크롤바 (ScrollAxis 방향): RecycleScrollbar
  │   │   ├─ Del = this (IRecycleScrollbarDelegate)
  │   │   ├─ 루프 스크롤 지원
  │   │   └─ LoadData 연동 (셀 재활용 기반 사이즈 계산)
  │   │
  │   └─ 보조축 스크롤바 (반대 방향): RecycleScrollbar (일반 모드)
  │       ├─ Content/Viewport 비율 기반 단순 동작
  │       ├─ 루프/LoadData 무관
  │       └─ 기존 ScrollRect 보조축 동작과 동일
  │
  ├─ 기존 RecycleScroller 기능 전체 유지
  │   ├─ 셀 재활용/풀링
  │   ├─ 루프 스크롤
  │   ├─ 페이지네이션
  │   ├─ 비동기 로드
  │   └─ 셀 그룹 배치
  │
  └─ LoadData 미호출 시
      └─ Content/Viewport 비율로 기본 스크롤+스크롤바 동작
          (주축/보조축 모두 동일한 방식)
```

### 구현 계획 (단계별)

#### Phase 1: ScrollRect 코드 포크 및 내재화
1. `RecycleScroller_ScrollRect.cs` 신규 생성 (partial class)
   - ScrollRect 소스 코드에서 핵심 로직 추출:
     - 필드: `m_content`, `m_viewport`, `m_horizontal`, `m_vertical`, `m_movementType`, `m_elasticity`, `m_inertia`, `m_decelerationRate`, `m_scrollSensitivity`, `m_velocity`
     - 드래그 핸들링: `OnBeginDrag`, `OnDrag`, `OnEndDrag`, `OnInitializePotentialDrag`
     - 스크롤 물리: `LateUpdate`의 관성/탄성/클램프/언리스트릭티드 계산
     - 위치 계산: `normalizedPosition`, `SetContentAnchoredPosition`
     - 이벤트: `onValueChanged`
   - `IBeginDragHandler`, `IDragHandler`, `IEndDragHandler`를 기존 `RecycleScroller`와 통합

2. `RecycleScroller_Inspector.cs` 수정
   - ScrollRect에서 가져온 직렬화 필드 추가
   - 기존 `m_scrollbarRef` (단일) → `m_horizontalScrollbarRef`, `m_verticalScrollbarRef` (듀얼)

3. `RecycleScroller.cs` 수정
   - `[RequireComponent(typeof(ScrollRect))]` 제거
   - `m_scrollRect` / `_ScrollRect` 필드/프로퍼티 제거
   - `Viewport`, `Content` 프로퍼티를 내재화 필드 참조로 변경
   - `RealNormalizedScrollPosition` getter/setter를 내부 계산으로 대체

#### Phase 2: `_ScrollRect.` 참조 제거
1. `RecycleScroller_Functions.cs` — 15곳의 `_ScrollRect.` 참조를 직접 필드 접근으로 교체
   - `UpdateScrollAxisToScrollRect()` → 내부 `m_horizontal`/`m_vertical` 설정
   - `OnScrollRectScrolling()` → `m_velocity` 직접 접근
   - `StartPagingCor()` → `m_inertia` 직접 접근
   - `NullifyScrollRectScrollbar()` → 삭제 (불필요)
2. `RecycleScroller_OnValidate.cs` — ScrollRect 없이 동작하도록 수정
3. Editor 파일 수정

#### Phase 3: 듀얼 스크롤바 지원
1. **주축 스크롤바** (ScrollAxis 방향): RecycleScrollbar 풀 기능
   - IRecycleScrollbarDelegate 연동, 루프 지원
   - LoadData 호출 시: 셀 재활용 기반 사이즈 계산
   - LoadData 미호출 시: Content/Viewport 비율 기반 (Phase 4와 동일)
2. **보조축 스크롤바** (반대 방향): RecycleScrollbar 일반 모드
   - Content/Viewport 비율 기반 단순 동작
   - 루프/LoadData 무관 — 항상 Content rect 크기로 동작
   - 기존 ScrollRect 보조축 스크롤바와 동일한 방식
3. 스크롤바 자동 바인딩 로직

#### Phase 4: LoadData 미호출 기본 동작
1. ScrollRect처럼 Content/Viewport 비율 기반 기본 스크롤 동작
2. 스크롤바 자동 크기 조절 (기존 `UpdateScrollbarSizeFromRect` 확장)
3. 스크롤 시 스크롤바 value 연동
4. 주축/보조축 모두 동일한 Content/Viewport 비율 방식

#### Phase 5: 마이그레이션
1. 기존 프리팹에서 ScrollRect 컴포넌트 제거
2. RecycleScroller에 viewport/content 참조 이전
3. **자동 마이그레이션 에디터 스크립트 제공** (`Tools > RecycleScroller > Migrate from ScrollRect`)
   - 프로젝트 전체 스캔: ScrollRect + RecycleScroller 공존하는 GameObject 탐지
   - ScrollRect의 viewport/content/movementType/elasticity 등 설정값을 RecycleScroller로 자동 이전
   - onValueChanged 프리팹 바인딩 정리
   - ScrollRect 컴포넌트 자동 제거
   - 변경 내역 로그 출력
