### 1. 클래스 선언 변경

#### 현재
```csharp
[RequireComponent(typeof(ScrollRect))]
[DisallowMultipleComponent]
[AddComponentMenu("UI/Recycle Scroll/Recycle Scroller")]
public partial class RecycleScroller : MonoBehaviour,
    IBeginDragHandler, IEndDragHandler, IRecycleScrollbarDelegate
```

#### 변경 후
```csharp
[DisallowMultipleComponent]
[ExecuteAlways]
[RequireComponent(typeof(RectTransform))]
[AddComponentMenu("UI/Recycle Scroll/Recycle Scroller")]
public partial class RecycleScroller : UIBehaviour,
    IBeginDragHandler, IDragHandler, IEndDragHandler,
    IInitializePotentialDragHandler, IScrollHandler,
    ICanvasElement, ILayoutElement, ILayoutGroup,
    IRecycleScrollbarDelegate
```

**변경 근거**:
- `MonoBehaviour` → `UIBehaviour`: ScrollRect와 동일. `OnRectTransformDimensionsChange` 등 UI 라이프사이클 훅 필요. `UIBehaviour`는 `MonoBehaviour`의 서브클래스이므로 기존 코드 100% 호환.
- `IDragHandler` 추가: ScrollRect의 `OnDrag` (매 프레임 드래그 위치 추적) 내재화
- `IInitializePotentialDragHandler` 추가: 드래그 시작 전 초기화 (ScrollRect 원본)
- `IScrollHandler` 추가: 마우스 휠 스크롤 (Gate-1 필수 보완)
- `ICanvasElement`: Canvas rebuild 참여 (`Rebuild`, `LayoutComplete`, `GraphicUpdateComplete`)
- `ILayoutElement`, `ILayoutGroup`: 레이아웃 시스템 참여 (스크롤바 auto-hide & viewport expand)
- `[RequireComponent(typeof(ScrollRect))]` 제거
- `[ExecuteAlways]` 추가: 에디터 모드 지원 (기존 OnValidate + Start 동작 유지)

**`RealNormalizedScrollPosition` 변경**:
```csharp
public float RealNormalizedScrollPosition
{
    get => ScrollAxis switch
    {
        eScrollAxis.VERTICAL => 1f - verticalNormalizedPosition,
        eScrollAxis.HORIZONTAL => horizontalNormalizedPosition,
        _ => 0f,
    };
    set
    {
        var setValue = value;
        if (m_loopScrollable)
        {
            if (setValue is < 0f or > 1f) setValue %= 1f;
            if (setValue < 0f) setValue += 1f;
        }
        switch (ScrollAxis)
        {
            case eScrollAxis.VERTICAL:
                SetNormalizedPosition(1f - setValue, 1);
                break;
            case eScrollAxis.HORIZONTAL:
                SetNormalizedPosition(setValue, 0);
                break;
        }
    }
}

### 3. OverwriteValue<MovementType> 적용 시점 (Gate-1 보완 #2)

| 시점 | 동작 |
|------|------|
| `OnValidate` (에디터) | `m_loopScroll` 체크 → Overwrite/RemoveOverwrite |

### 4. 듀얼 스크롤바 아키텍처

```
RecycleScroller
├─ 인스펙터 필드
│   ├─ m_verticalScrollbar: RecycleScrollbar (수직 스크롤바)
│   └─ m_horizontalScrollbar: RecycleScrollbar (수평 스크롤바)

├─ 주축 스크롤바 (MainAxisScrollbar — ScrollAxis 기반 자동 결정)

│   ├─ LoadData 호출 후: RecycleScroller가 Size/Value 직접 관리
│   ├─ LoadData 미호출 시: Content/Viewport 비율 기반 Size, LateUpdate에서 value 동기화
│   ├─ 루프 모드 지원 (LoopScrollbarMode)
│   └─ ScrollbarVisibility 적용 (Permanent/AutoHide/AutoHideAndExpandViewport)
│
└─ 보조축 스크롤바 (CrossAxisScrollbar — ScrollAxis 기반 자동 결정)

### 5. LoadData 미호출 기본 동작

LoadData가 호출되지 않은 상태 (`m_cellCount == 0`)에서의 동작:

1. **스크롤 물리**: `LateUpdate`의 ScrollRect 포크 로직이 그대로 동작
   - Content의 anchoredPosition을 드래그/관성/탄성으로 이동
   - `UpdateBounds`가 Content/Viewport의 실제 Rect로 Bounds 계산

2. **주축 스크롤바**: `LateUpdate`에서 `OnScrollPositionChanged` 호출 시:
   - `m_cellCount == 0`이면 셀 재활용 로직 스킵 (`UpdateCellView` 내부 가드)
   - 스크롤바 value만 `SetScrollbarValueWithoutNotify`로 동기화
   - 스크롤바 size는 `Start`/`OnValidate`에서 `UpdateScrollbarSizeFromRect`로 설정 (기존 구현)

3. **보조축 스크롤바**: `UpdateCrossAxisScrollbar`가 `LateUpdate`에서 항상 동작

4. **`OnScrollPositionChanged` 내부 가드**:
```csharp
private void OnScrollPositionChanged(Vector2 val)
{
    // 루프 reposition (LoadData 후에만 동작)
    if (m_cellCount > 0 && m_scrollerMode.NeedReposition(...))
    {
        var vel = m_velocity;
        ShowingScrollPosition = ShowingScrollPosition;
        m_velocity = vel;
    }

    // 스크롤바 value 동기화 (항상 동작)
    SetScrollbarValueWithoutNotify();

    // 셀 업데이트 (LoadData 후에만 동작)
    if (m_cellCount > 0)
        UpdateCellView();

    // 사용자 콜백 (항상 동작)
    onScroll?.Invoke(val);
    // ... 방향 변경, 페이지 변경 등
}
```

### 6. OnEnable/OnDisable 통합

```csharp
// RecycleScroller_ScrollRect.cs

protected override void OnEnable()
{
    base.OnEnable();

    // ScrollAxis 기반 m_horizontal/m_vertical 초기화
    UpdateScrollAxisToScrollRect();

    // 보조축 스크롤바 강제 비활성화
    HideCrossAxisScrollbar();

    // Canvas rebuild 등록
    CanvasUpdateRegistry.RegisterCanvasElementForLayoutRebuild(this);
    SetDirty();
}

private void Start()
{
    // OverwriteValue<MovementType> 초기 적용
    // (OnValidate는 Play 모드에서 호출 안 되므로, LoadData 전까지의 공백 방지)
    if (m_loopScroll)
        m_overwriteMovementType.Overwrite(ScrollRect.MovementType.Unrestricted);

    UpdateScrollbarSizeFromRect();
}

protected override void OnDisable()
{
    // 1. Canvas rebuild 해제
    CanvasUpdateRegistry.UnRegisterCanvasElementForRebuild(this);

    // 2. 상태 초기화
    m_dragging = false;
    m_scrolling = false;
    m_hasRebuiltLayout = false;
    m_velocity = Vector2.zero;

    // 3. 기존 RecycleScroller OnDisable 로직
    StopAllMoveCor();
    StopAllPreviousLoadDataTask();
    StopCorWaitLoadDataStateToCompleteForBuffer();

    // 4. Layout 정리
    m_tracker.Clear();
    LayoutRebuilder.MarkLayoutForRebuild(_RectTransform);
    base.OnDisable();
}
```

### 9. ScrollRect 기능 포크 여부

| ScrollRect 기능 | 포크 여부 | 이유 |
|----------------|----------|------|
| `ScrollbarVisibility` (Permanent/AutoHide/AutoHideAndExpandViewport) | **포함** | 주축 스크롤바에 3가지 모드 모두 지원. RecycleScrollbar 기반으로 재작성 |
| `UpdateScrollbarVisibility` | **포함** | ScrollbarVisibility 구현에 필요. RecycleScrollbar의 gameObject.SetActive()로 구현 |
| `UpdateScrollbarLayout` (viewport 자동 리사이즈) | **포함** | AutoHideAndExpandViewport 구현에 필요. 주축 스크롤바 숨김 시 보조축 방향 viewport 확장 (스크롤 축 ViewportSize에 영향 없음 → 셀 재활용 무관) |
| `m_HSliderExpand` / `m_VSliderExpand` | **포함** | UpdateScrollbarLayout에서 사용. 내부 런타임 플래그 |
| `ILayoutGroup` (`SetLayoutHorizontal`, `SetLayoutVertical`) | **포함** | viewport auto-expand 동작에 필수. ScrollRect 원본 로직 활용 |
| `m_HorizontalScrollbar` / `m_VerticalScrollbar` (`Scrollbar` 타입) | **제외** | `RecycleScrollbar`로 대체. Unity `Scrollbar` 타입 바인딩 불필요 |

**보조축 스크롤바**: 코드에서 강제 비활성화 (`gameObject.SetActive(false)`). RecycleScroller 설계상 보조축 Content 길이가 항상 viewport에 맞춰지므로 스크롤 자체가 불필요. 에디터 인스펙터에서 HelpBox로 안내.
