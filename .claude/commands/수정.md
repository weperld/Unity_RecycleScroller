## 사전 준비

1. PROJECT_SUMMARY.md를 읽고 프로젝트 컨텍스트를 파악하세요
2. WORK_IN_PROGRESS.md를 확인하여 충돌하는 작업이 없는지 확인하세요
3. 수정 대상 파일을 반드시 읽고 문제를 분석하세요
4. WorkID를 생성하세요 (형식: WIP-YYYYMMDD-NNN)
5. WORK_IN_PROGRESS.md에 새 작업을 등록하세요

## 파이프라인

유형A(수정)로 판단하여 진행합니다.

```
Plan → Design → Code → Test → Docs → QA → Review
```

### 각 단계 수행 방법

**Plan**: 문제 분석, 영향 파일 식별, 수정 범위 확정
- `.wips/templates/WIP-Plan-YYYYMMDD-NNN.md` 참조하여 WIP 작성
- 수렴 검증 수행: 누락·모호·위험 요소가 0건이 될 때까지 반복
- 사용자 확인 후 다음 단계

**Design**: 수정 설계 (필요 시)
- 단순 버그 수정은 간략히, 구조 변경 시 상세 설계
- 수렴 검증 수행
- 사용자 확인 후 다음 단계

**Code**: 구현
- CLAUDE.md의 절대 규칙 및 Unity/C# 규칙 준수
- 코드 스타일: `.guides/CODE_STYLE.md` 참조
- 기술 규칙: `.guides/TECHNICAL_RULES.md` 참조

**Test → Docs → QA → Review**: 신규.md와 동일

### Gate 검증

신규.md의 Gate 검증 테이블과 동일하게 적용합니다.
Gate 3회 실패 시 이전 단계로 롤백합니다.
