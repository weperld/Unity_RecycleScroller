## 파이프라인

아래 순서대로 진행합니다. 각 단계 완료 시 Gate 검증을 수행하고, 사용자 피드백을 받은 후 다음 단계로 넘어갑니다.

```
Plan → Design → Code → Test → Docs → QA → Review
```

### 각 단계 수행 방법

**Plan**: 요구사항 분석, 영향 파일 식별, 작업 범위 확정
- `.wips/templates/WIP-Plan-YYYYMMDD-NNN.md` 참조하여 WIP 작성
- 수렴 검증 수행: 누락·모호·위험 요소가 0건이 될 때까지 반복
- 사용자 확인 후 다음 단계

**Design**: 아키텍처 설계, 클래스 구조, 인터페이스 정의
- `.wips/templates/WIP-Design-YYYYMMDD-NNN.md` 참조
- 수렴 검증 수행
- 사용자 확인 후 다음 단계

**Code**: 구현
- `.wips/templates/WIP-Code-YYYYMMDD-NNN.md` 참조
- CLAUDE.md의 절대 규칙 및 Unity/C# 규칙 준수
- 코드 스타일: `.guides/CODE_STYLE.md` 참조
- 기술 규칙: `.guides/TECHNICAL_RULES.md` 참조

**Test**: 테스트 작성 및 실행
- `.wips/templates/WIP-Test-YYYYMMDD-NNN.md` 참조
- 테스트 가이드: `.guides/TEST_GUIDE.md` 참조

**Docs**: 문서 업데이트
- `.wips/templates/WIP-Docs-YYYYMMDD-NNN.md` 참조

**QA**: 품질 검증
- `.wips/templates/WIP-QA-YYYYMMDD-NNN.md` 참조

**Review**: 최종 리뷰 및 완료 처리

### Gate 검증 (각 단계 전이 시)

| 전이 | 크로스체크 담당 | 검증 내용 |
|------|--------------|----------|
| Plan → Design | architect | 요구사항 완전성, 영향 분석 정확성 |
| Design → Code | developer | 설계 구현 가능성, 인터페이스 명확성 |
| Code → Test | tester | 컴파일 통과, 절대 규칙 준수 |
| Test → Docs | developer | 테스트 커버리지, 버그 0건 |
| Docs → QA | reviewer | 문서 완전성 |
| QA → Review | architect | 전체 품질 |

Gate 3회 실패 시 이전 단계로 롤백합니다.

## 완료 검증 (코드 변경 시 필수)

1. **컴파일 검증**: 에러 0건
2. **검증 항목**: `.guides/VERIFICATION_ITEMS.md` 체크리스트 확인
3. **Self-Validation Checklist**: AGENTS.md의 체크리스트 수행 및 결과 보고
4. **문제 발생 시**: `.guides/VERIFICATION_ITEMS.md`에 검증 항목 추가
